# Kivira outbound message pattern (v6, 2026-05-20)

Execution variant used for the Wave 1 **10-person in-scope** follow-up cohort after Josh review. Builds on [[KIVIRA_OUTBOUND_MESSAGE_PATTERN_2026_05_12]] (v5) with sender-visible tweaks from live sends.

[VERIFIED: Shape documented in `messages-josh-preview.md` §Working assumptions (v6) and per-person drafts (2026-05-20).]

## The shape

```
{First name},

Currently helping build out go-to-market for Kivira.health, mental-health clinical decision
support for primary care, backed by Antler.

{1–2 sentences: role-specific value-prop tied to sub-tier pain.}

Open to a short exploratory call? If your week's packed, point me to whoever at {company}
owns this and I'll take it there. Our founder Maria would join for the demo, and we'd
genuinely value your feedback.

Keegan
```

## Changes from v5

| Element | v5 | v6 |
|---------|----|----|
| Opener | No greeting; "Inside Kivira.health:" | **First name + comma**; GTM lead-in sentence |
| Social proof | Antler + Wellstar Catalyst | **Antler only** (Wellstar optional / dropped for length) |
| Demo line | "Demo would be with our founder." | **"Our founder Maria would join for the demo"** |
| Close | "Your read on this, or direction toward whoever…" | Exploratory call **or** point to owner; feedback framing |

## Style guardrails (v6)

- No em-dashes; periods and commas only.
- No "Hi" opener; no thanks-for-connect line in the follow-up DM.
- No "folks" — use role titles or "leaders" / "clinicians."
- Claims-safe: drafts intentionally omit unverified funding figures and unnamed healthcare-system backers (see preview note block).

## Evidence

> "First name + comma. No 'Hi' opener." — `messages-josh-preview.md`, Working assumptions (v6)

## Execution learnings (same cohort)

- **Broken copy** on first send (Lystrup) required apology + clean resend — template QA before send.
- **First booked demo** (Oltmans) followed by **operator no-show** — execution discipline, not copy, became the bottleneck ([[WEEKLY_HEYREACH_EVIDENCE_2026_05_20]]).
