# Demo-first outbound strategy

[INFERRED: Early-stage outbound should optimize for learning and booked demos before investing in scale infrastructure (domains/inboxes/automation).]

## Operating rule

- **Default**: run manual multi-channel outreach first (LinkedIn + phone + personalized email).
- **Upgrade to scaled infrastructure** only after you can show repeatable signal (reply rate, meetings booked) tied to a defined ICP/subtier + message.

## Why this exists

[INFERRED: “Infrastructure” (inboxes/domains/sequencing) is primarily a throughput unlock; it does not create product-market narrative or ICP clarity by itself.]

## Practical implications

- **Start with the smallest viable loop**: pick one subtier, build a small list (few personas per account), run outreach, measure outcomes.
- **Avoid premature complexity**: don’t let deliverability tooling distract from message testing and learning “why people buy.”
- **Scale when you have a working unit**: once one subtier/message is working, replicate to additional subtiers/campaigns.

## Notes on scaling

[INFERRED: Multiple inboxes/domains are mainly required to scale volume safely; they matter after the message/targeting works, not before.]
